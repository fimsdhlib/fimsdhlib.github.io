For this tool review, I have chosen a data management system I have seen often in the Digital Humanities world: what is described as “WordPress for collections” (Cohen 2011): Omeka.

### 1) A short description/context of the tool, where it originated, how it was created and by whom, development status
Omeka is a digital publishing platform first developed at the [Roy Rosenzweig Center for History and New Media (CHNM)](https://rrchnm.org/) at George Mason University in Virginia in 2008. Since 2016 it has been taken on by the [Corporation for Digital Scholarship](https://digitalscholar.org/), a non-profit which curates multiple digital projects. Omeka is a digital curation tool, designed as a virtual exhibition space for projects in an interactive set of displays. When it was launched (Omeka is a Swahili word meaning to “lay out wares”), “few options existed for museums, libraries, and archives wishing to publish collections and narrative exhibits to the web as easily as one could launch a blog” (“Project”). The developers at CHNM noted existing options were not cross-compatible or flexible enough to be user-friendly and effective, and launched what is now known as Omeka Classic. In 2017, the project launched Omeka S, a collaborative version of the platform designed to work from a central pool of resources across multiple projects. Both Omeka Classic and Omeka S co-exist as options for digital creation.

### 2) A short description of what the tool is meant to do; what doors does it open for scholarly pursuits; and if it is unique in that regard
Omeka is an open source digital exhibition space, with the intention that projects may be hosted with relative ease and built within a series of themes for content hosting and positive UX. The tool can be used to host exhibits, collections, and interactive maps, with flexible layouts and a strong focus on narrative building. Case studies include collections dedicated to a core group (the [Lomax Kentucky Recordings collect audio](https://lomaxky.omeka.net/) recordings from rural Kentucky between 1933 and 1942), theme (the [Humboldt Redwoods Project](https://hsuredwoodsproject.omeka.net/) examines local relationship with redwood forests), and region (the [Hermoupolis Digital Heritage Management](https://hermoupolis.omeka.net/) project geotags photos and documents within a single Greek municipality).

The uniqueness of this project came from necessity: the project was founded to respond to a need for a collections management system that was not only accessible for users but responsive to programmers for maintenance. As such, Omeka trades on its user experience on the front and back end, with responsive analytics and simple themes and templates for hosting materials. Here, collections can be hosted in a list format, tagged and populated with metadata, and arranged as needed. As much or as little of the collection can be outwardly displayed as required, meaning new items can be seamlessly added as needed. For scholarly pursuits, Omeka fills a strong niche, as the interactivity of the collection becomes key to later scholarly output such as publications and analysis, and allows supplementary work (particularly with Omeka-S sharing datasets).

The alternatives to Omeka fall short on the UX front: WordPress offers options for collection management, but there is a steep learning curve in the back end, and is not as responsive in its flexibility with collections. Many other alternatives to Omeka are “mainly Learning Management Systems but may also be CMS Tools or Online Education Services” (“Omeka Alternatives”), but the easy-to-use installation and quick setup is a major benefit. While other apps like DEVCommunity, Chamilo, Instructure Canvas, and Koha do compete with Omeka, the great benefit here is that users do not need specific programming skills to manage the system. 

### 3) A critical review of its function, considering elements such as: functionality and accessibility; performance; learning curve; documentation and additional support material; application; possible use cases (in the classroom or for research).

The functionality of Omeka is very straightforward. Users can sign up for a free trial site which gives the opportunity to test the tool prior to committing to the full package. Once a site is created–which allows for URL personalisation and site title—collections are added on a tiered scale, logically broken down as “item,” “collection,” and “exhibit.”  Items are the structural core of the site, and users may upload individual artifacts or objects with a great deal of metadata right off the bat. Each item bears (optional) fields for Title, Subject, Description, Creator, Source, Date, Contributor, Rights, Format, Language, Type, Identifier, and Coverage, all of which can be rendered in HTML and which features a handy Dublin Core metadata set common to all Omeka fields. Uploading files is very straightforward (each individual site is allowed 500MB storage in the trial package, and 2GB/5GB/10GB/50GB on each successive pricing tier) and can be easily toggled between public and private. Items are organised as “collections,” which helps to group objects by theme or location, and the overarching organisational principle of “exhibit” houses the collections.

Omeka has a very user-friendly dashboard which features search functions, reports (with outputs that include atom, dcmes-xml, json, omeka-xml, rss2), and theme toggling, which makes this an extremely satisfying tool to build through. This requires nearly no technical expertise, but also has options for editing code if so required. The site offers Plugins which are fantastic to extend the use of the site with specialised tools for mapping and timelines, for example. These plugins work much in the same way as those from WordPress, but installing and using them feels much less finicky. There are also options to change settings in terms of query types and indexing, making this feel extremely responsive and apt for personalisation.

The site also features excellent support, with strong FAQ pages and a hugely useful set of step by step guides which demonstrate the tool’s features and its hidden benefits, meaning any new user has a wide slate of options for developing the right resource for them.

The use cases are extremely varied: Omeka offers a terrific [“use cases” page](https://info.omeka.net/use-cases/) which demonstrates how the resource can be used by Archivists, Educators, Enthusiasts, Librarians, Museum Professionals, and Scholars. The resources Omeka hosts could easily be used in both practical and archival settings, for displaying a collection for browsing or for research and data analysis. 

This is an outstanding resource which will appeal to the neophyte and seasoned curator alike: reasonably priced, easy to use and attractive in its product.
